# Фотопроекты

Эта папка содержит документальные фотопроекты для блога.

## Структура фотопроекта

Каждый фотопроект - это файл Markdown с YAML front matter. Структура выглядит так:

```yaml
---
layout: photoproject
title: "Название проекта"
description: "Краткое описание проекта"
date: 2025-01-20
location: "Место съемки"
cover_image: "cover.jpg"  # опционально, для обложки в списке проектов
images:
  - path: "image1.jpg"
    caption: "Подпись к фотографии"
  - path: "image2.jpg"
    caption: "Еще одна подпись"
---
```

## Поля front matter

- **layout**: всегда `photoproject`
- **title**: название проекта (обязательно)
- **description**: краткое описание для списка проектов (обязательно)
- **date**: дата проекта в формате YYYY-MM-DD (обязательно)
- **location**: место съемки (опционально)
- **cover_image**: имя файла обложки для превью в списке (опционально)
- **images**: массив объектов с фотографиями (обязательно)
  - **path**: имя файла фотографии
  - **caption**: подпись к фотографии (опционально)

## Структура папок

Для каждого фотопроекта создайте папку с изображениями:

```
assets/images/photoprojects/
  └── название-проекта/
      ├── cover.jpg          # обложка (опционально)
      ├── image1.jpg
      ├── image2.jpg
      └── image3.jpg
```

Название папки должно совпадать с именем файла проекта (без расширения `.md`).

## Пример

1. Создайте файл `_photoprojects/my-project.md`
2. Создайте папку `assets/images/photoprojects/my-project/`
3. Добавьте фотографии в папку
4. Заполните front matter в файле проекта

## Возможности

- Полноэкранный просмотр фотографий (lightbox)
- Навигация стрелками клавиатуры
- Подписи к фотографиям
- Адаптивный дизайн
- Плавные переходы и анимации
- Навигация между проектами

## Рекомендации

- Используйте фотографии высокого качества
- Оптимизируйте размер файлов для быстрой загрузки
- Добавляйте описательные подписи к фотографиям
- Используйте осмысленные названия файлов
