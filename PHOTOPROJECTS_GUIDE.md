# Руководство по работе с фотопроектами

## Обзор

Система фотопроектов позволяет создавать документальные фотопроекты и визуальные истории в стиле профессиональных фотографов.

## Основные возможности

- ✅ Галерея фотографий с полноэкранным просмотром (lightbox)
- ✅ Навигация клавиатурой (стрелки влево/вправо, Escape)
- ✅ Подписи к фотографиям
- ✅ Адаптивный дизайн для всех устройств
- ✅ Поддержка светлой и темной темы
- ✅ Навигация между проектами
- ✅ Минималистичный дизайн в стиле документальной фотографии

## Создание нового фотопроекта

### Шаг 1: Создайте файл проекта

Создайте файл в папке `_photoprojects/` с именем проекта, например:

```
_photoprojects/my-project.md
```

### Шаг 2: Создайте папку для изображений

Создайте папку для изображений в `assets/images/photoprojects/`:

```
assets/images/photoprojects/my-project/
```

**Важно:** Название папки должно совпадать с именем файла проекта (без расширения `.md`).

### Шаг 3: Добавьте изображения

Поместите фотографии в папку проекта:

```
assets/images/photoprojects/my-project/
  ├── cover.jpg          # обложка для списка проектов (опционально)
  ├── image1.jpg
  ├── image2.jpg
  └── image3.jpg
```

### Шаг 4: Заполните front matter

Откройте файл проекта и заполните YAML front matter:

```yaml
---
layout: photoproject
title: "Название проекта"
description: "Краткое описание проекта для списка"
date: 2025-01-20
location: "Место съемки"
cover_image: "cover.jpg"  # опционально
images:
  - path: "image1.jpg"
    caption: "Подпись к первой фотографии"
  - path: "image2.jpg"
    caption: "Подпись ко второй фотографии"
  - path: "image3.jpg"
    caption: "Подпись к третьей фотографии"
---
```

### Шаг 5: Добавьте вводный текст

После front matter добавьте вводный текст к проекту:

```markdown
---

Это вводный текст к фотопроекту. Здесь вы можете описать контекст, идею проекта или историю, которую хотите рассказать через фотографии.

Вы можете использовать **маркдаун** для форматирования:

- Списки
- **Жирный текст**
- _Курсив_

После вводного текста будут отображены все фотографии из массива `images`.
```

## Структура файла проекта

### Обязательные поля

- `layout`: всегда `photoproject`
- `title`: название проекта
- `description`: краткое описание для списка проектов
- `date`: дата проекта в формате YYYY-MM-DD
- `images`: массив объектов с фотографиями

### Опциональные поля

- `location`: место съемки
- `cover_image`: имя файла обложки для превью в списке
- `caption`: подпись к каждой фотографии (в массиве images)

## Пример полного файла проекта

```yaml
---
layout: photoproject
title: "Городские пейзажи"
description: "Серия фотографий городской жизни"
date: 2025-01-20
location: "Москва, Россия"
cover_image: "cover.jpg"
images:
  - path: "street1.jpg"
    caption: "Утренняя улица"
  - path: "street2.jpg"
    caption: "Вечерний свет"
  - path: "street3.jpg"
    caption: "Городская жизнь"
---

Это фотопроект о городских пейзажах. Я хотел запечатлеть жизнь города в разное время суток.

Проект показывает контраст между дневной и ночной жизнью мегаполиса.
```

## Использование lightbox

- **Клик по фотографии**: открывает полноэкранный просмотр
- **Стрелка влево**: предыдущее изображение
- **Стрелка вправо**: следующее изображение
- **Escape**: закрыть lightbox
- **Клик по фону**: закрыть lightbox

## Рекомендации

1. **Оптимизация изображений**: Используйте оптимизированные изображения для быстрой загрузки
2. **Размер файлов**: Рекомендуемый размер каждого изображения - не более 2-3 МБ
3. **Формат**: Используйте JPEG для фотографий, PNG для графики
4. **Разрешение**: Рекомендуемое разрешение - не более 2000px по длинной стороне
5. **Подписи**: Добавляйте осмысленные подписи к фотографиям
6. **Названия файлов**: Используйте описательные названия файлов

## Структура проекта

```
blog/
  ├── _photoprojects/              # Файлы проектов
  │   ├── my-project.md
  │   └── another-project.md
  ├── assets/
  │   └── images/
  │       └── photoprojects/
  │           ├── my-project/
  │           │   ├── cover.jpg
  │           │   ├── image1.jpg
  │           │   └── image2.jpg
  │           └── another-project/
  │               └── ...
  ├── photoprojects.html           # Страница со списком проектов
  └── _layouts/
      └── photoproject.html        # Layout для проектов
```

## Доступ к проектам

- **Список проектов**: `/photoprojects/`
- **Отдельный проект**: `/photoprojects/имя-проекта/`

## Удаление проекта

Чтобы удалить проект:

1. Удалите файл из `_photoprojects/`
2. Удалите папку с изображениями из `assets/images/photoprojects/`

## Поддержка

Если у вас возникли вопросы или проблемы, проверьте:

1. Правильность структуры front matter
2. Совпадение названий файла проекта и папки с изображениями
3. Корректность путей к изображениям
4. Формат даты (YYYY-MM-DD)

## Дополнительные возможности

### Добавление обложки

Для добавления обложки проекта в списке проектов:

1. Добавьте изображение `cover.jpg` в папку проекта
2. Укажите `cover_image: "cover.jpg"` в front matter

Если обложка не указана, будет использовано первое изображение из массива `images`.

### Сортировка проектов

Проекты автоматически сортируются по дате (новые первыми). Чтобы изменить порядок, измените дату в front matter.

### Навигация между проектами

На каждой странице проекта есть навигация для перехода к предыдущему и следующему проекту.

## Стилизация

Все стили находятся в `assets/css/style.css`. Вы можете настроить:

- Размеры изображений
- Отступы между фотографиями
- Стили подписей
- Цвета и шрифты
- Анимации и переходы

## Технические детали

- Используется Jekyll Collections для организации проектов
- Lightbox реализован на чистом JavaScript
- Поддержка lazy loading для оптимизации загрузки
- Адаптивный дизайн с CSS Grid и Flexbox
- Поддержка светлой и темной темы
